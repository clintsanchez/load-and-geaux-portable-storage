## Workflow & Best Practices

### Content Creation Workflow
1. **Plan** — Review content calendar and keyword clusters in `/blog/clusters/`
2. **Research** — Use materials in `/docs/research/` and existing strategy docs
3. **Draft** — Create in `/blog/drafts/` with proper frontmatter
4. **Review** — Check brand tone (brand-style-guide.md) and SEO guidelines
5. **Publish** — Move to `/blog/published/` and deploy to website
6. **Promote** — Add to social calendar and email campaigns

### Design & Graphics Workflow
1. **Approve** — Check `/assets/` for existing graphics
2. **Adapt** — Use Stitch templates from `/brand/` for consistency
3. **Generate** — Create new assets if needed (Figma or Claude Design)
4. **Optimize** — Ensure brand compliance and web optimization
5. **Store** — Add to appropriate `/assets/` subfolder

### Code Commits
- **Branch naming:** `feature/feature-name` or `content/article-name`
- **Commit messages:** Clear, descriptive, referencing related issues
- **Pull requests:** Include context, related issues, and testing notes
- **Merge:** Squash commits for cleaner history

### Code Review Standards
- All PRs require review before merge
- Check brand compliance and style guide adherence
- Verify links, images, and formatting
- Test locally before approval

### Brand Compliance
- ⚠️ **NO clip art or hand-drawn SVGs** — Use AI-generated images from client's actual photos (via Gemini image gen)
- ✅ **Use pre-built assets** — 40+ testimonial cards, 12+ infographics ready in `/assets/`

### Design Tools
- **Source of Truth:** GitHub repository is authoritative
- **Stitch + Claude Design:** Repo is source; Claude Design has no API (one-way GitHub re-sync)

- ⚠️ **Hard gate:** Every homepage/landing page must pass 16-element conversion checklist
- 📋 Check list: CTA buttons, value props, social proof, forms, optimization metrics
- No silent omissions — explicitly document any deviations
